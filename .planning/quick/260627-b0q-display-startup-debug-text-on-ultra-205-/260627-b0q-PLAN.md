# Quick Task 260627-b0q: Display startup debug text on Ultra 205 OLED

Date: 2026-06-27

## Goal

Display a minimal startup-only debug screen on the Ultra 205 SSD1306 OLED while preserving the current fail-closed safe boot behavior.

## Tasks

1. Add a pure startup debug text model in `bitaxe-core`.
   - Files: `crates/bitaxe-core/src/lib.rs`
   - Action: add a typed formatter for the four OLED lines and tests proving each line fits the 128x32 screen with the selected small font geometry.
   - Verify: `cargo test -p bitaxe-core startup_debug --all-features`
   - Done: four exact text lines are generated from board, ASIC, safe state, and firmware commit inputs.

2. Add and wire the firmware display adapter.
   - Files: `Cargo.toml`, `firmware/bitaxe/Cargo.toml`, `firmware/bitaxe/src/main.rs`, `firmware/bitaxe/src/display_adapter.rs`, `firmware/bitaxe/src/asic_adapter.rs`
   - Action: add `ssd1306` and `embedded-graphics`, render the startup lines over I2C0 SDA GPIO47/SCL GPIO48 at address `0x3c`, and refactor `Peripherals::take()` so firmware startup claims peripherals once.
   - Verify: firmware build and package commands complete.
   - Done: display failures log `display_status=unavailable ...` and do not abort boot; successful render logs `display_status=startup_text_rendered ...`.

3. Record parity and task evidence.
   - Files: `docs/parity/checklist.md`, `docs/parity/evidence/ultra-205-startup-display-debug-2026-06-27.md`, `.planning/STATE.md`, quick-task summary.
   - Action: mark display/I2C rows in progress, add evidence with command output and hardware status, and update GSD state.
   - Verify: `just parity`
   - Done: docs describe implementation evidence without overclaiming hardware verification if flashing is not run.

## Acceptance

- The firmware attempts startup display rendering before the ASIC boot gate.
- The OLED text is exactly:
  - `Bitaxe Rust`
  - `Ultra 205 BM1366`
  - `SAFE no mining`
  - `fw <12-char commit>` or `fw Unavailable`
- Mining, ASIC work submission, and hardware control remain disabled.
