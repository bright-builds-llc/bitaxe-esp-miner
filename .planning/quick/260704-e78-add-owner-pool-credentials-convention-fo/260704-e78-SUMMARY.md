# Summary

Added a repo-local convention for owner-supplied live mining pool input through ignored `pool-credentials*.env` files.

## Changes

- Added `.gitignore` coverage for `pool-credentials*.env`.
- Documented the expected local env fields: `BITAXE_POOL_URL`, `BITAXE_POOL_PORT`, `BITAXE_POOL_USER`, and `BITAXE_POOL_PASSWORD`.
- Restricted agents to sourcing those files only as local runtime input for repo-owned live mining tests.
- Kept raw owner address, worker string, endpoint, port, password, target URL, IP, MAC, Wi-Fi, token, and NVS secret values out of committed evidence.

## Verification

- `git check-ignore pool-credentials.test.env`
- `rg "pool-credentials|BITAXE_POOL|DEVICE_URL" AGENTS.md .gitignore`
- Phase 21 wrapper missing-credentials check
- Phase 21 wrapper sourced-credentials redaction check
- `git diff --check`
- `cargo fmt --all`
- `cargo clippy --all-targets --all-features -- -D warnings`
- `cargo build --all-targets --all-features`
- `cargo test --all-features`

## Residual Risk

- The convention does not create or store a real owner BTC address. Live evidence still requires the operator-owned local env file, a valid same-session target, redaction review, and the existing hardware safety gates.
