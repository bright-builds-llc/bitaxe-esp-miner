<threat_model>
## Trust Boundaries

| Boundary | Description |
| --- | --- |
| LAN client -> firmware HTTP/WebSocket server | Untrusted clients can read logs/state or request settings/command changes. |
| HTTP settings route -> ESP-IDF NVS | Validated settings become persistent device configuration. |
| HTTP command route -> restart/display/mining state effects | API requests can affect visible device state or restart timing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-05-15 | Spoofing / Elevation of Privilege | all routes | mitigate | Enforce private-network/AP-origin gate on HTTP and WebSocket handshakes; tests deny rejected origins. |
| T-05-16 | Tampering / Denial of Service | PATCH settings | mitigate | Body cap enforced before JSON parsing or persistence, structured JSON parse, pure all-or-nothing validation, NVS write/commit/reload before success, and tests proving oversized bodies return generic public errors with zero writes. |
| T-05-17 | Denial of Service | WebSocket server | mitigate | Max 10 clients, no-client hibernation, unchanged telemetry no-send, bounded log chunks. |
| T-05-18 | Repudiation | command routes | mitigate | Typed internal logs for accepted/rejected commands without leaking secrets. |
| T-05-19 | Safety impact | pause/resume/restart/identify/OTA | mitigate | Commands cannot bypass mining gates; restart response precedes effect; OTA/OTAWWW return safe unsupported/deferred failure. |

High-risk block: stop if any settings, command, log, or WebSocket route is reachable without the access gate; if denied HTTP/WebSocket requests do not return the planned public denial shape; if oversized settings PATCH bodies can write, commit, reload, or expose rich public errors; if restart can occur before the response is sent; or if OTA/OTAWWW can apply an update in Phase 5.
</threat_model>
