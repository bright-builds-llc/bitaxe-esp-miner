<threat_model>
## Trust Boundaries

| Boundary | Description |
| --- | --- |
| Observation collection to safety core | Runtime or evidence observations become typed prerequisite inputs. |
| Safety core to mining loop | Pure readiness decisions determine whether BM1366 work dispatch can proceed. |
| Mining runtime to API output | Blocker reasons become user-visible and potentially evidence-cited strings. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-22-04 | Elevation of Privilege / Tampering | `MiningLoopGate` | mitigate | Require `ProductionMiningPreconditionDecision::Ready` before dispatch and forbid force/override/bypass flags in touched files. |
| T-22-05 | Tampering | `BoundedObservationEvidence` | mitigate | Require source, board `205`, evidence id, validity window, and reason before bounded evidence can satisfy a prerequisite. |
| T-22-06 | Information Disclosure | Blocker reason strings and API `blockedReason` | mitigate | Use stable generic snake_case reason strings only; do not include pool credentials, device URLs, IPs, MACs, Wi-Fi values, NVS secrets, raw Stratum payloads, raw share payloads, or raw BM1366 frames. |
| T-22-07 | Spoofing / Tampering | Runtime observations | mitigate | Treat stale, unavailable, unsafe, ambiguous, undocumented, or board-mismatched observations as blocked decisions that keep work submission disabled. |
</threat_model>
