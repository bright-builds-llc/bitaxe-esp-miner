## Trust Boundaries

| Boundary | Description |
| --- | --- |
| parsed ASIC result -> submit intent | Parsed nonce observations become submit intents only through active-work correlation. |
| Phase 22 safety gate -> production dispatch | Production dispatch must stay disabled when prerequisite readiness is blocked. |
| pure core -> firmware logs | Firmware sees typed statuses and must not log raw ASIC or pool values. |
| raw-bearing submit structs -> formatter/log surfaces | Submit intent and share submission data carry job id, extranonce, nonce, version, target, and share context that must not leak through generic formatting. |
| guarded core -> controlled firmware runtime | The controlled runtime must consume production dispatch/result outputs and keep diagnostics separate from production routing. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
