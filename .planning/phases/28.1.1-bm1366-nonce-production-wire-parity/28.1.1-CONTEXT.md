## Specific Ideas

- The key distinction is transmitted intent versus accepted hashing state: Rust can emit matching commands while the BM1366 remains UART-responsive but electrically idle.
- A safe readback can be `unavailable`; unavailable state should route to lifecycle comparison rather than inventing a register value or raw write.
- A fake-pool correlate/submit is a strong discriminator but does not replace the original live accepted/rejected share gate for Phase 30 promotion.
- The OS-native reader is authoritative only after it proves nonzero application delivery; node appearance, stable identity, ownership, firmware production, and observed bytes remain separate facts.
- Replay proves current-session boot facts after enumeration; it does not fabricate original pre-enumeration bytes.
