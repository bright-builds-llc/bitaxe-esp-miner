## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| In-memory snapshot -> reload decisions | Stored values may be missing, legacy, invalid, or corrupt. |
| Unit/golden evidence -> parity checklist | Evidence type can be overstated if pure tests are treated as hardware proof. |
| Config persistence decisions -> future firmware adapter | Later adapter code will consume these typed operations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering / Elevation of Privilege | `persistence.rs` snapshot writes | mitigate | Persistence applies typed `NvsWrite` operations to an in-memory map only and grep checks prove no ESP-IDF or hardware calls exist. |
| T-02-02 | Information Disclosure | Evidence and fixtures | mitigate | Evidence doc states fixtures use upstream public defaults only and adds no Wi-Fi or private pool credentials. |
| T-02-03 | Tampering / Denial of Service | Snapshot key reload | mitigate | Reload uses Plan 02 exact key names and migration rules; tests prove active plus legacy keys roundtrip. |
| T-02-04 | Denial of Service | Corrupt float reload | mitigate | Persistence test proves corrupt `asicfrequency_f` reloads to default `485` instead of failing load. |
| T-02-05 | Spoofing / Safety governance failure | `docs/parity/checklist.md` CFG rows | mitigate | Checklist updates explicitly keep non-205 boards not hardware-verified and forbid hardware evidence claims from pure tests. |
</threat_model>
