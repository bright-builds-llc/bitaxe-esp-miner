Create `crates/bitaxe-safety/src/observation.rs` as a pure module. Define public newtypes `BootSessionId(u64)`, `ObservationSequence(u64)`, and `MonotonicMillis(u64)` with documented constructors/accessors and no wall-clock conversion. Define `StampedSample<T> { value, boot_session, sequence, acquired_at }`, `StaleReason`, `UnavailableReason`, and `FaultReason` as serde-stable enums using lower-snake-case serialization. Define `Observation<T>` with exact variants `Fresh { sample }`, `Stale { last_good, reason }`, `Unavailable { reason }`, and `Fault { reason, maybe_last_good }`; optional fields and locals use the required `maybe_` prefix.

Add pure producer transition functions or methods with these exact rules: a successful validated input creates `Fresh` and advances the supplied source-local sequence exactly once; `mark_stale` requires an existing last-good sample and preserves every stamp field; `record_fault` never stores the failed attempted value and retains the prior last-good sample when present; `Unavailable` contains no sample. Add helpers `maybe_last_good()`, `state_label()`, and `is_fresh()` without any clock or I/O reads. Cross-boot ordering must return an explicit unordered/different-session result rather than comparing raw sequences.

Write Arrange/Act/Assert unit tests covering every variant, success advancement, stale preservation, fault with/without last-good, unavailable absence, same-session sequence ordering, cross-session rejection, and serde labels. Export the module from `lib.rs` and add `src/observation.rs` to the Bazel library srcs. Run the focused test and keep the implementation generic; do not add API wire fields or firmware storage in this task.
