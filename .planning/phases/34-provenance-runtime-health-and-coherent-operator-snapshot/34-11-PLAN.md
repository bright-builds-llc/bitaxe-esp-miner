<layout-authority>
The implementation must preserve Plan 34-10's complete supported-image policy and add only these two invariants from pinned local ESP-IDF v5.5.4 authority:

1. `.embuild/espressif/esp-idf/v5.5.4/components/bootloader_support/src/esp_image_format.c` states that `esp_app_desc_t` is in DROM and always in application segment 0, and it reads descriptor fields from the first bytes of that segment. `.embuild/espressif/esp-idf/v5.5.4/components/bootloader_support/src/bootloader_utility.c::bootloader_common_get_partition_description` likewise locates the descriptor immediately after the 24-byte image header and first 8-byte segment header. `.embuild/espressif/esp-idf/v5.5.4/components/esp_app_format/esp_app_desc.c` emits the descriptor into `.rodata_desc`; `.embuild/espressif/esp-idf/v5.5.4/components/esp_system/ld/esp32s3/sections.ld.in` places `.rodata_desc` first inside `.flash.appdesc` and says nothing may precede it; `memory.ld.in` asserts the flash rodata dummy begins at the rodata-segment origin. Therefore application segment 0 must classify as DROM, must be non-empty and large enough for the complete descriptor, and the descriptor magic/fields must begin at payload offset zero. Existing DROM mapped congruence is still checked independently; do not scan later bytes or later segments for a descriptor, and do not accept a different memory family merely because its payload contains authentic descriptor bytes.
2. Every non-empty segment already has a checked `load_addr..load_addr + data_len` range inside one admitted family. Before entry or descriptor validation, compare every pair with half-open overlap semantics `left.start < right.end && right.start < left.end`; equality at either boundary is valid adjacency. Zero-length segments remain accepted under Plan 34-10 semantics, still undergo mapped congruence when classified DROM/IROM, but create no destination range and never participate in overlap checks. Then check the ESP32-S3 shared D/IRAM physical view using pinned `soc.h`: `SOC_DIRAM_IRAM_LOW = 0x40378000`, `SOC_DIRAM_DRAM_LOW = 0x3fc88000`, and `SOC_I_D_OFFSET = 0x006f0000`. Normalize each non-empty IRAM interval into the DRAM coordinate space with checked subtraction of `0x006f0000` and compare it to every non-empty DRAM interval; this is conservative for the admitted IRAM-only prefix because it maps below the admitted DRAM floor. Direct numeric overlap must be classified before alias overlap so stable categories are deterministic. Do not conflate destination overlap with flash payload overlap, DROM/IROM MMU congruence, entry containment, or bootloader runtime checks outside the schema-v3 authority.

Construct a private `ValidatedSegmentLayout` (or equivalently strong type) that owns the parsed segments and their already-checked non-empty destination ranges only after descriptor-segment, direct-overlap, and alias-overlap validation succeeds. `ValidatedEsp32S3Image` must own that layout rather than remaining a copyable count/entry summary. Entry and descriptor validation must accept only the validated layout, so unchecked segment vectors cannot reach them. Keep the implementation dependency-free and avoid quadratic complexity beyond the maximum 16 segments.
</layout-authority>
