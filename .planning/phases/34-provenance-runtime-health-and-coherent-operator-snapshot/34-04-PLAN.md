  <action>
Create a dependency-free functional core in `runtime_health.rs` with these exact serialized vocabularies:

- `PassiveSelfTestState`: `idle`, `blocked`, `running`, `passed`, `failed`, `canceled`, `unavailable`.
- `SupervisorAvailability`: `available`, `unavailable`.
- `CheckpointHealth`: `healthy`, `stale`, `unhealthy`, `unavailable`.
- `TaskWatchdogParticipation`: `participating`, `not_participating`, `unavailable`; Phase 34 must produce `unavailable` with reason `unproved` unless an existing read-only source directly proves one of the other values.

Represent a checkpoint observation with bounded category text, `sequence: u64`, and producer-owned `observed_at_millis: u64`. Reject an empty category, a category longer than 32 ASCII characters, sequence regression, monotonic-time regression, or checked-arithmetic overflow. Do not invent a checkpoint when none exists. Derive `checkpoint_age_millis` from the current monotonic time at each operator-snapshot publication. Define the health thresholds from the existing publisher cadence: `stale_after_millis = 3 * publish_interval_millis` and `unhealthy_after_millis = 10 * publish_interval_millis`, using checked multiplication. Ages at or below the stale threshold are `healthy`; ages above stale and at or below unhealthy are `stale`; ages above unhealthy are `unhealthy`. Invalid/missing timing inputs are `unavailable`, never healthy.

Add `RuntimeHealthSnapshot` containing the passive self-test state, supervisor availability, optional bounded checkpoint category/sequence/age, checkpoint health, and the independent task-watchdog participation fact plus nullable reason. Keep all constructors pure and require callers to supply already-observed state and monotonic timestamps; expose no callback, command, mutable device handle, or start/feed/configure method.

Create `runtime_health_adapter.rs` as an imperative-shell reader only. It may copy existing self-test lifecycle and supervisor/checkpoint observations into the pure evaluator. When an existing producer cannot prove a value, pass `unavailable`; do not create a self-test task or mode, register/feed/configure ESP task watchdog, sleep/spin to manufacture age, inject load/faults, or call GPIO, I2C writes, reset, power, fan, voltage, ASIC, pool, Stratum, or mining code.

Extend the Phase 34 coherent snapshot in `runtime_snapshot.rs` so runtime health is captured once per existing boot-session/operator-snapshot revision. Project that same captured value into the shared system-info and live-WebSocket wire model as additive `runtimeHealth`. Emit one redacted retained record per published revision with `boot_session`, `operator_snapshot_revision`, the bounded lifecycle/availability/category/sequence/age/health/watchdog values, and `redacted=true`; do not include credentials, network identity, targets, device identifiers, or secrets. Do not alter heartbeat, PATCH, restart, or unrelated response shapes.

Add Arrange/Act/Assert tests covering all seven self-test states; direct serialization spelling; missing observations; category bounds; sequence/time regression and overflow; exact healthy/stale/unhealthy boundary values; repeated publications whose time advances while sequence remains fixed; recovery only after sequence advances; watchdog `unavailable/unproved` remaining distinct from an available healthy supervisor; identical runtime-health values and correlation fields in system-info, WebSocket, and retained projections; and preservation of unrelated public fixtures. Add a source-guard Bazel test that scans the new core/adapter/snapshot path and fails on active self-test, `esp_task_wdt_` mutation, load/fault injection, GPIO/I2C write, reset/power/fan/voltage/ASIC/mining imports or calls.
