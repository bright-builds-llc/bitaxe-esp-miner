# Phase 28.1 - Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

## Test Infrastructure

| Property | Value |
| --- | --- |
| **Framework** | Cargo/Bazel `rust_test` over `crates/*` only — firmware-crate `#[cfg(test)]` tests do not execute on host (esp-idf-sys gate; see 26-03-SUMMARY.md). Firmware changes verify by `just build` xtensa compile + hardware markers. |
| **Config file** | `Cargo.toml` workspace; `BUILD.bazel` per touched crate (new `bridge_orchestration.rs` added to explicit Bazel `srcs` in Plan 01 Task 2). |
| **Quick run command** | `cargo test -p bitaxe-stratum -p bitaxe-asic` |
| **Full suite command** | `cargo fmt --all && cargo clippy --all-targets --all-features -- -D warnings && cargo build --all-targets --all-features && cargo test --all-features && just test` |
| **Estimated runtime** | Quick under 60 seconds; full gate a few minutes (Bazel cache warm). Hardware captures 360 s + wall clock >= 420 s. |

## Sampling Rate

- **After every task commit:** Run `cargo test -p bitaxe-stratum -p bitaxe-asic`
- **After every plan wave:** Run the full suite command (plus `just build` for firmware plans 02-04)
- **Before `/gsd-verify-work`:** Full suite green plus hardware evidence ladder captured (Plan 05)
- **Max feedback latency:** 420 seconds (hardware flash-monitor runs; host tests under 60 s)
