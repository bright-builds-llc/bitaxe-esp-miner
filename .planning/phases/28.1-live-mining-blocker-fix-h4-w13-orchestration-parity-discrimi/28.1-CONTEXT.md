<domain>
## Phase Boundary

Fix the v1.1 milestone blocker: the Ultra 205 BM1366 never returns a UART byte after work dispatch (no nonces, no shares), despite chip detect, byte-matched init, matched TX framing, and an active live Stratum session (Phase 27 outcome; deferred in `phase27-04-defer-note.md`).

This phase delivers:
1. Upstream-parity mining orchestration as the DEFAULT behavior of the Phase 27 live-hardware-bridge mode (H4 + W13 closure).
2. Discriminating hardware probes (post-init register read, INA260 power delta) that split the remaining search space if nonces still do not appear.
3. Detector-gated hardware evidence runs against the connected Ultra 205, with an approved upstream stock-firmware A/B fallback.

Out of scope: non-205 boards, other ASIC families, Stratum v2, share-difficulty target validation beyond existing behavior, checklist promotion beyond what hardware evidence supports.

Root-cause facts established during investigation (treat as given, do not re-derive):
- Protocol port is faithful: init frames byte-match upstream fixtures; CRC5/CRC16 equivalent to upstream `crc.c`; PLL search, DS4432U voltage math, difficulty-mask math all match upstream; ticket mask from difficulty 1000 → 511 is sane.
- Defect 1 (J2c confound): in `firmware/bitaxe/src/live_stratum_runtime.rs:run_asic_bridge_step` (~line 779), `awaiting_result_read` is checked BEFORE `maybe_pending_dispatch`; with the continuous listener armed, dispatch is unreachable. The only H4 hardware run (J2c) never dispatched work, so H4 has never actually been tested.
- Defect 2 (W13 boot gate): the default phase27 package requires a nonce from synthetic boot diagnostic work within 10 s or production peripherals are never retained (`production_ready()` stays false → `ASIC_INITIALIZED_GATE_MISSING`). Upstream sets `ASIC_initalized=true` after init unconditionally.
- Defect 3: `job_redispatch_pump` re-dispatches nothing — it sets a pending flag but `dispatch_next()` dequeues from an empty queue; there is no work regeneration with fresh extranonce2 (upstream regenerates every ~2 s in `create_jobs_task`).

## Implementation Decisions

### Workstream A — Upstream-parity orchestration as phase27 default
- New pure orchestrator `crates/bitaxe-stratum/src/v1/bridge_orchestration.rs`: `BridgeOrchestrator` with injected monotonic clock; `next_step()` encodes dispatch-before-poll priority (fixes the J2c bug), 2 s regeneration cadence, timeout→continue (never fatal), session invalidation clears the redispatch timer.
- `crates/bitaxe-stratum/src/v1/live_runtime.rs`: store `maybe_current_notify` + `extranonce2_counter`; new `regenerate_work()` rebuilds work with `.with_extranonce2_value(counter++)` and stride-8 job id; counter resets on clean_jobs/reconnect/authorization reset/session replacement. First dispatch stays extranonce2=0 for golden-fixture parity.
- `crates/bitaxe-asic/src/bm1366/mining_ready.rs`: `bm1366_job_interval_ms(asic_count)` = `2000 / next_power_of_two(asic_count)` mirroring upstream `ASIC_get_asic_job_frequency_ms`; retire firmware-side `JOB_REDISPATCH_INTERVAL_MS`.
- W13 boot semantics (`firmware/bitaxe/src/asic_adapter.rs`, `asic_adapter/work_result_investigation.rs`): after chip detect + mining-ready init succeed in `Phase27ProductionBridge` mode, skip boot diagnostic work and retain production peripherals unconditionally, publishing `InitializedNoMining`. Keep `require_diagnostic_nonce`/`require_uart_proof_for_production` as opt-out rollback levers. Power/thermal/ack safety gates untouched.
- Pump rewiring (`firmware/bitaxe/src/live_stratum_runtime.rs`): reorder `run_asic_bridge_step` (dispatch first, then ≤100 ms result poll slice); continuous listener + redispatch pump become default (delegate `AsicBridgeState` to `BridgeOrchestrator`); remove fail-closed `ResultTimeout` from the default pump (timeout = continue + telemetry); keep mining after first accepted share (replace NormalStop exit); safe-stop/cleanup paths untouched. Single `phase27-bridge` thread retained — no second thread.
- Telemetry: keep marker strings comparable with J-runs (`work_dispatched`, `rx_chunk`, `h4_continuous_result=*`, `result_correlated`); add `asic_bridge=job_redispatched extranonce2_counter=<n>` and `asic_bridge=timeout_streak count=<n>`. Run docs must note `production_result_timeout` disappears by design.
- Flag disposition (last, own commit): retire `continuous_result_task`, `job_redispatch_pump`, `initialized_no_mining_gate`, `skip_boot_diagnostic_work`; add `single_dispatch_bounded_read` opt-out for A/B; keep `clear_rx_before_production_work` opt-in. Fail-closed default build (no compile envs → no mining) unchanged.

### Workstream B — Discriminating hardware probes
- Post-init register read probe: after full init + baud switch (and again after work dispatch), read reg 0x00 (chip id, like upstream `BM1366_read_registers`) and log outcome. Response → chip alive, RX path fine at 1 MBaud → problem is nonce production. Silent → chip wedged by init or 1 M baud domain broken.
- INA260 power-delta marker: sample power immediately before and ~5–10 s after first dispatch; log delta. Hashing at 485 MHz ⇒ large jump (~10 W+); flat ⇒ cores not hashing.
- Evidence must confirm bring-up logs `safety_voltage_effect=write setpoint_v=1.200` matches `defaults.asic_voltage_mv()`.

### Workstream C — Hardware evidence runs
- `just detect-ultra205` gate first; build via `scripts/phase27-live-hardware-bridge-package.sh` (no `BITAXE_WORK_RESULT_INVESTIGATION`); flash-monitor with `capture-timeout-seconds=360`, wall clock ≥420 s; credential files per standing permission; redaction rules per CLAUDE.md.
- Success ladder: `initialized_no_mining` → bridge active → multiple `work_dispatched` ~2 s apart (falsifies J2c confound by itself) → register-read probe response → `rx_chunk`/`result_correlated` → `share_submission_status=accepted`.
- Control run with `single_dispatch_bounded_read` to confirm old behavior reachable (regression guard).
- Upstream A/B fallback (USER APPROVED): if probes show chip-alive-but-no-nonces or chip-silent, flash stock upstream ESP-Miner (factory image or locally built from `reference/esp-miner` pin) on the Ultra 205. Recovery path (documented for phase gating): reflash Rust firmware via `just flash-monitor board=205 ...` + NVS reseed. If upstream mines → hardware healthy; optionally build upstream with `BM1366_SERIALTX_DEBUG`/RX debug to capture real wire bytes and diff against Rust output. If upstream does not mine → board/ASIC health problem; stop firmware work and report.
