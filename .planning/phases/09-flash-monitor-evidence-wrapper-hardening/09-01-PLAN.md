<threat_model>
## Trust Boundaries

| Boundary | Description |
| --- | --- |
| CLI input to process execution | User-supplied `board`, `port`, image, manifest, evidence directory, and capture timeout become process arguments and filesystem paths. |
| USB serial output to evidence JSON | Serial monitor bytes from a connected device become committed evidence after classification. |
| Long-running monitor process to local automation | `espflash monitor` is continuous by design and must not hang `just flash-monitor` evidence capture. |
| Failure output to operator | Monitor failures produce recovery instructions that influence the next hardware action. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-09-01-01 | Tampering | `CommandSpec` process execution | mitigate | Continue using `Command::new("espflash")` with argument vectors; do not build shell command strings or use shell `timeout`. |
| T-09-01-02 | Tampering | `monitor_log_has_trusted_boot_markers` | mitigate | Require all seven serial-scope boot markers before recording a passing capture conclusion. |
| T-09-01-03 | Denial of Service | `LocalFlashEnvironment::execute_capturing` | mitigate | Enforce `DEFAULT_MONITOR_CAPTURE_TIMEOUT_SECONDS = 25` through Rust-owned spawn/try_wait/kill/wait flow and expose `capture-timeout-seconds` for slow boots. |
| T-09-01-04 | Repudiation | `flash-command-evidence.json` | mitigate | Record separate `flash_command`, `monitor_command`, `capture_status`, `trusted_output`, timeout, log path, commits, manifest path, and conclusion fields. |
| T-09-01-05 | Information Disclosure | Monitor evidence | mitigate | This plan adds no network probes or secret-bearing config reads; evidence fields are board, port, commits, manifest/image paths, commands, capture status, and serial boot markers only. |
| T-09-01-06 | Elevation of Privilege | Recovery guidance | mitigate | Guidance points to `just detect-ultra205`, wrapper `just flash-monitor`, and diagnostic `just monitor`; it does not instruct operators to use raw `espflash monitor` as proof. |
</threat_model>
