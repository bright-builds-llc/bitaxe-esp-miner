<threat_model>
## Trust Boundaries

| Boundary | Description |
| --- | --- |
| pool notify/difficulty -> production work registry | Parsed pool job context enters trusted BM1366 work tracking. |
| clean-jobs/reconnect -> active work | Session-changing events must invalidate stale queued and active work. |
| registry target context -> logs/evidence | Difficulty and target context must not render raw targets in committed artifacts. |
| raw-bearing production structs -> formatter/log surfaces | Work records and dispatches carry job ids, extranonce, target context, and payload context that must never leak through generic formatting. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-24-05 | Tampering / Replay | `ProductionWorkRegistry` | mitigate | Session generation increments on clean-jobs, reconnect, authorization reset, and pool-session replacement. |
| T-24-06 | Spoofing | active work lookup | mitigate | `maybe_active_work` returns only records from the current generation. |
| T-24-07 | Information Disclosure | `ProductionTargetContext` | mitigate | Target context keeps internal numeric fields and does not render raw Stratum target strings. |
| T-24-08 | Denial of Service | bounded queue | mitigate | Registry reuses the existing bounded queue and returns existing queue errors instead of unbounded allocation. |
| T-24-18 | Information Disclosure | `ProductionWorkRecord` / `ProductionDispatch` | mitigate | Raw-bearing registry structs avoid derived Debug/display or use custom redacted formatting tests that reject job id, extranonce, target, payload, nonce, and share context leakage. |
