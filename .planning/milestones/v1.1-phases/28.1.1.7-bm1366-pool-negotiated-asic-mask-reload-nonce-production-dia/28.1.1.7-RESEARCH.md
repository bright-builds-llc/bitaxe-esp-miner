**Key insight:** For BM1366, pool-negotiated mask reload is a **separate UART register write after configure**, not a job-byte or MiningWork-field change. Fake-poolÔÇÖs identical mask value makes **reload TX observation** the structural proof; hashing may still stay idle.

## Common Pitfalls

### Pitfall 1: Fake-Pool Identical Mask Blind Spot (CRITICAL)
**What goes wrong:** Plan requires mask-value delta or rising `power_delta` to call A/B success; concludes ÔÇťreload didnÔÇÖt matterÔÇŁ because value equals init.
**Why it happens:** Fake pool returns `1fffe000` = `STRATUM_DEFAULT_VERSION_MASK` / init hardcoded mask.
**How to avoid:** Gate lever success on `mask_reload_tx_observed` + `post_configure_runtime` (D-05). Treat unchanged hashing after observed reload as valid sole-blocker falsification toward next_hypothesis (D-06).
**Warning signs:** Disposition cites ÔÇťmask unchangedÔÇŁ without reload TX class.

### Pitfall 2: Counting Init Frames as Runtime Reload
**What goes wrong:** Comparator sets `mask_reload_tx_observed: true` from prelude ├Ś3 / init register.
**Why it happens:** All SetVersionMask frames share the same wire encoding.
**How to avoid:** Require explicit `post_configure_runtime` marker after configure; unit-test fixtures with init-only logs Ôćĺ observed false (D-08/D-11).
**Warning signs:** Pre-patch Wave 0 baseline already shows reload observed.

### Pitfall 3: Reloading Before `production_ready()`
**What goes wrong:** TX attempted on non-retained UART / before mining-ready; silent failure.
**Why it happens:** Configure can conceptually race init; upstream gates on `ASIC_initalized`.
**How to avoid:** Pending flag until `asic_adapter::production_ready()`; flush before/at first dispatch (create_jobs semantics).
**Warning signs:** Marker never appears despite mask_stored.

### Pitfall 4: Patching Job Bytes or Reopening Work-Field Sole-Blocker
**What goes wrong:** OR mask into version field or re-run work-field A/B as primary lever.
**Why it happens:** Confusion with 28.1.1.6.
**How to avoid:** HARD BAN `negotiated_version_mask_work_field_parity` in recommender; retain mask-on-work wire flag only (D-09/D-15).
**Warning signs:** Job version class leaves `base_notify`.

### Pitfall 5: Second Speculative Patch
**What goes wrong:** After reload TX + idle cores, implement midstate/frequency/etc. in same phase.
**Why it happens:** Upstream create_jobs has many adjacent surfaces.
**How to avoid:** Disposition only sets evidence-named `next_hypothesis` (D-06).
**Warning signs:** PR touches mining.rs job layout + production reload together without disposition gate.

### Pitfall 6: Dropping Retained Wire Parity
**What goes wrong:** A/B reverts ASIC-256, RX-loop, or mask-on-MiningWork.
**Why it happens:** Unrelated init churn.
**How to avoid:** Assert retention flags in evidence (D-15).
**Warning signs:** Evidence omits `wire_parity_*_retained`.

### Pitfall 7: Early Exit Before 360 s / Missing Detector Gate
**What goes wrong:** False negative on correlate/submit.
**Why it happens:** Default monitor capture is 25 s.
**How to avoid:** `capture-timeout-seconds>=360`, wall clock Ôëą420 s, `just detect-ultra205` first (D-16).
**Warning signs:** Evidence `capture_timeout_seconds` < 360.
