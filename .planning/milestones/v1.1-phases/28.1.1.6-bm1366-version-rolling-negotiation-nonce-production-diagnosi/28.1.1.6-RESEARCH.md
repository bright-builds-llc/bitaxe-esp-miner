## HOW to apply negotiated mask to BM1366 work fields (Plan 02 critical)

Upstream semantics for Ultra 205 / BM1366 (do not invent a new encoding):

1. **Job UART `version` bytes = base notify version only.**  
   `construct_bm_job` sets `new_job->version = params->version` and never ORs the mask into that field. `BM1366_send_work` copies that base version into the 4-byte job `version` slot and forces `num_midstates = 0x01`. [VERIFIED: `mining.c:71`; `bm1366.c:315-321`; `bm1366.h` `BM1366_job`]

2. **Persist negotiated mask on the work object.**  
   Upstream assigns `next_job->version_mask = GLOBAL_STATE->version_mask` after construct. Rust must stop discarding `maybe_version_mask` and store it on `MiningWork` (new field, e.g. `maybe_version_mask: Option<VersionMask>`). [VERIFIED: `create_jobs_task.c:213-217`]

3. **Host midstate path (pure-crate fixtures; not BM1366 UART).**  
   When `version_mask != 0`, upstream builds midstate0 from base version, then midstate1..3 by `increment_bitmask(version, mask)` three times (`mining.c:95-111`; algorithm at `mining.c:162-179`; fixture vectors in `test_mining.c:108-120`). BM1366 **does not transmit** those midstate arrays. Plan 02 may add pure helpers + fixtures mirroring that host behavior for parity evidence, but must **not** change `Bm1366WorkPayload` layout or set `num_midstates` ≠ `0x01`. [VERIFIED: `work.rs:13,69-80`; wire-diff-summary `num_midstates`]

4. **ASIC register programming is separate (out of scope this phase).**  
   Chip rolling window is `versions_to_roll = version_mask >> 13` written to reg `0xA4` (`BM1366_set_version_mask` / Rust `version_mask_payload`). Init already programs `STRATUM_DEFAULT_VERSION_MASK` / `0x1fffe000`. Pool-driven reload is upstream `create_jobs_task.c:126-129` — that is the fail-closed **next** hypothesis, not Plan 02. [VERIFIED: `bm1366.c:141-147`; `registers.rs:102-111`; `utils.h` `STRATUM_DEFAULT_VERSION_MASK`; D-05/D-14]

5. **Submit `version_bits`.**  
   Upstream: `rolled_version = job.version | version_bits_from_asic`; submit `version_bits = rolled_version ^ job.version`. Rust already submits ASIC-parsed `version_bits` (`<< 13` from result frame). Keep that path; do not invent a second encoding. [VERIFIED: `bm1366.c:368-377`; `asic_result_task.c:64,108-116`; `result.rs:168-177`]

**Plan 02 one-liner:** Remove the discard; attach mask to `MiningWork`; keep `Bm1366WorkFields.version == notify.version` LE bytes; fixture-pin midstate/`increment_bitmask` host semantics; do not patch job bytes or ASIC reload.

## Common Pitfalls

### Pitfall 1: Masking the Job Version Field
**What goes wrong:** Plan ORs `version \| mask` (or similar) into `Bm1366WorkFields.version`.
**Why it happens:** Misreading `construct_bm_job` midstate loop as wire encoding.
**How to avoid:** Fixture asserts `fields.version == notify.version.to_le_bytes()` with nonzero mask; cite `BM1366_send_work`.
**Warning signs:** Diffs that change the 4 version bytes in job payload tests.

### Pitfall 2: Claiming 11/11 Already Covers Version Rolling
**What goes wrong:** Skip work-field patch because job bytes match.
**Why it happens:** Comparator only checked base notify version.
**How to avoid:** Emit `mask_applied_to_work` separately from job version match (D-02/D-07).
**Warning signs:** Disposition says “already matched” without `mask_applied_to_work: true`.

### Pitfall 3: Implementing ASIC Reload as Second Patch
**What goes wrong:** Scope creep after work-field A/B unchanged.
**Why it happens:** Upstream `create_jobs_task` reload is the obvious next diverge.
**How to avoid:** Disposition only sets `next_hypothesis: pool_negotiated_mask_asic_reload` (D-05).
**Warning signs:** `SetVersionMask` calls added outside init in the same PR as mining.rs.

### Pitfall 4: Reopening Falsified Levers
**What goes wrong:** Diagnostic recommends poll/long-block/ticket-mask/enumerate RX-loop.
**Why it happens:** Older recommenders still list those enums.
**How to avoid:** Hard-fail unit tests if banned labels appear (D-08).
**Warning signs:** `recommended_investigation` contains any HARD BAN string.

### Pitfall 5: Fake-Pool Mask Equals Init Default Blind Spot
**What goes wrong:** Expect hashing to change when mask value is identical (`1fffe000`).
**Why it happens:** Fake pool fixture matches `STRATUM_DEFAULT_VERSION_MASK`.
**How to avoid:** Still require `mask_applied_to_work` evidence; treat unchanged hashing as valid falsification toward ASIC-reload hypothesis (timing/order of reload may still matter upstream even when value matches).
**Warning signs:** Plan invents a different fake-pool mask without documenting why.

### Pitfall 6: Dropping Retained Wire Parity
**What goes wrong:** A/B reverts ASIC-256 or RX-loop.
**Why it happens:** Unrelated init churn.
**How to avoid:** Assert `wire_parity_ticket_mask_retained` + `wire_parity_rx_loop_retained` in evidence (D-14).
**Warning signs:** Evidence omits retention flags.

### Pitfall 7: Early Exit Before 360 s / Missing Detector Gate
**What goes wrong:** False negative on correlate/submit.
**Why it happens:** Default monitor capture is 25 s.
**How to avoid:** `capture-timeout-seconds>=360`, wall clock ≥420 s, `just detect-ultra205` first (D-15).
**Warning signs:** Evidence `capture_timeout_seconds` < 360.
