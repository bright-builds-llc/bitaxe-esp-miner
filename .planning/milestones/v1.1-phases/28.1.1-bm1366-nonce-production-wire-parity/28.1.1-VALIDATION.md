## Sampling Rate

- **After every task commit:** Run the task-scoped pure-crate or script test named by the plan.
- **After every plan wave:** Run all affected crate and script tests; hardware waves also rerun the detector and reference guard.
- **Before phase verification:** Run the full Rust suite, all new comparator/diagnostic tests, reference verification, lifecycle validation, and a redaction sweep.
- **Max host feedback latency:** 30 seconds for quick tests on a warm build cache.
- **Hardware feedback budget:** At least 360 seconds capture and at least 420 seconds shell wall-clock budget.

## Threat Model References

- **T-28.1.1-01 — Information disclosure:** raw pool, network, credential, register, or UART values escape into committed evidence. Mitigate with path-only inputs, closed-set summaries, and denylist/redaction tests.
- **T-28.1.1-02 — Evidence tampering/repudiation:** structural markers or fake-pool-only results are promoted as live mining proof. Mitigate with the exact D-18 gate and Phase 30 ownership.
- **T-28.1.1-03 — Unsafe hardware action:** ambiguous ports, undocumented raw writes, short captures, or missing recovery gates produce unsafe or meaningless evidence. Mitigate with detector, board-info, repo-owned commands, and bounded documented reads only.
- **T-28.1.1-04 — Observation interference:** diagnostic register reads recreate the falsified continuous polling behavior and alter the ASIC state being measured. Mitigate with bounded stage-bracketed snapshots and an explicit no-default-polling assertion.
