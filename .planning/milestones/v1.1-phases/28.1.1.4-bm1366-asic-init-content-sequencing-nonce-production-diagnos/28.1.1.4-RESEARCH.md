## Common Pitfalls

### Pitfall 1: Comparing Against Self-Derived Fixtures Instead of Live SERIALTX
**What goes wrong:** Planner assumes dynamic frames already match (as 28.1.1 summary claimed) and skips the ticket-mask fix.
**Why it happens:** Host tests assert Rust output equals `DIFFICULTY_1000_FRAME`, which is consistent with Rust's wrong input source.
**How to avoid:** Comparator must use upstream monitor `tx: [...]` frames; report category match vs upstream, not vs checked-in 1000 fixture alone.
**Warning signs:** `difficulty_mask_match: true` while upstream class is `diff_256` and Rust is `diff_1000`.

### Pitfall 2: Confusing Pool `stratumdiff` With ASIC Family Difficulty
**What goes wrong:** “Fix” keeps reading `primary_pool().difficulty()` (1000).
**Why it happens:** NVS `stratumdiff` and catalog naming both say “difficulty.”
**How to avoid:** Patch input to upstream `family.asic.difficulty` (=256 for BM1366). Add catalog field if missing rather than hard-coding silently without provenance comment.
**Warning signs:** Fixture still named/equal to `DIFFICULTY_1000_FRAME` after the “fix.”

### Pitfall 3: `1000` vs `512` Mask Collisions
**What goes wrong:** Classifier reports `diff_512` for Rust pool-1000 frames (identical wire bytes).
**Why it happens:** Power-of-two mask for 1000 and 512 both yield 511 → `00 00 80 FF`.
**How to avoid:** Prefer source-aware labels (`rust_source: pool_stratumdiff_1000`) plus wire class `mask_eq_pow2_le_512`.
**Warning signs:** Unit tests pass for 512 while product intent was 1000 or 256.

### Pitfall 4: Double Upstream Init Skews Counts
**What goes wrong:** `frequency_tx` 140 vs 70 looks like a ramp bug.
**Why it happens:** Upstream capture re-inits; each ramp ~70 PLL writes.
**How to avoid:** Compare final PLL + presence of ramp, not raw frequency_tx equality.
**Warning signs:** `frequency_final_match: true` but count mismatch flagged as failure.

### Pitfall 5: Reopening Falsified Levers
**What goes wrong:** Diagnostic recommends `post_max_baud_delay_2000`, `match_upstream_register_read_poll`, or `upstream_like_long_block_receive`.
**Why it happens:** Older recommenders still list those enums.
**How to avoid:** Closed recommendation set for 28.1.1.4; hard-fail tests if falsified labels appear.
**Warning signs:** `recommended_investigation` contains any falsified knob.

### Pitfall 6: Early Exit Before 360 s / Missing Detector Gate
**What goes wrong:** False negative on correlate/submit.
**Why it happens:** Default monitor capture is 25 s.
**How to avoid:** `capture-timeout-seconds>=360`, wall clock ≥420 s, `just detect-ultra205` first.
**Warning signs:** Evidence `capture_timeout_seconds` < 360.
