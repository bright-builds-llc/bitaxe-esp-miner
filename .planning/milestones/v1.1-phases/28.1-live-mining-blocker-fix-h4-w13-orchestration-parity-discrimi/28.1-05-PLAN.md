If J3 reaches nonces/shares, SKIP the A/B entirely.

**Procedure:**
1. Obtain upstream image WITHOUT dirtying the pin: primary — copy `reference/esp-miner` (pin `c1915b0a63bfabebdb95a515cedfee05146c1d50`) to the session scratchpad directory and build there with the upstream ESP-IDF toolchain; fallback — download the official bitaxeorg/ESP-Miner factory release image for Ultra (record exact release version and filename in the run doc). If neither path is achievable (missing upstream toolchain and no network image), record `ab_run: blocked-toolchain` in the disposition and stop — do not improvise.
2. Flash the upstream image to the Ultra 205 with `espflash` against the detector-selected port (record the exact command). Do NOT run any full-chip erase; record whether upstream flashing touched anything beyond app/OTA partitions.
3. Monitor >= 360 s: does upstream mine (ASIC responses/nonces/hashrate/share activity in upstream logs)? Record observed behavior with all pool/Wi-Fi/network material redacted to category labels.
4. **Recovery path (mandatory, executed in the same session):** reflash the Rust firmware full factory image via `just flash-monitor board=205 port=<detected-port> capture-timeout-seconds=360 wifi-credentials=wifi-credentials.json` — the existing repo-owned flow that rebuilds/repackages and reseeds NVS (prefer the full factory-image path per RESEARCH Assumption A3 so partial-partition assumptions are irrelevant). Verify recovery by observing trusted Rust boot markers (`phase27_safety_bring_up=complete`, `bootstrap=initialized_no_mining`) in the recovery capture. Wall clock >= 420 s.
5. Run `just verify-reference` — MUST pass (pin untouched by the out-of-tree build).

**Required evidence for the A/B section of the run doc:** board 205, selected port, Rust source commit, reference pin commit, upstream image provenance (pin build or release version), exact flash/monitor/recovery commands, board-info output, redacted observed behavior, recovery confirmation markers, conclusion.
