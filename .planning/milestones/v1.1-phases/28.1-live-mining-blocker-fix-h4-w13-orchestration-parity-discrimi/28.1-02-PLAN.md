<objective>
Make upstream-parity orchestration the DEFAULT behavior of the Phase 27 live-hardware-bridge firmware: W13 boot-gate removal (retain production peripherals unconditionally after successful init) and pump rewiring (dispatch-before-poll via BridgeOrchestrator, default continuous listener + regeneration cadence, timeout = continue, keep mining after first share).

Purpose: Upstream sets `ASIC_initalized = true` unconditionally after init (asic_init.c:61) and its result task never fails on timeout (asic_result_task.c). Our default package instead demands a diagnostic nonce in 10 s (never satisfied) and the bridge starves dispatch behind an armed listener. This plan closes both gaps as defaults, not investigation flags.

Output: Firmware compiles for xtensa (`just build` + phase27 package build); all orchestration decisions delegate to the host-tested `BridgeOrchestrator`; safety and safe-stop paths untouched. Firmware `#[cfg(test)]` additions are compile-contracts only — executable proof lives in Plan 01 tests and Plan 05 hardware evidence.
</objective>
