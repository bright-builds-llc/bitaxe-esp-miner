## Common Pitfalls

### Pitfall 1: Reopening the `0x0A` Frame Myth
**What goes wrong:** Plan patches ReadChipId payload “to match upstream.”
**Why it happens:** Older notes quote init3 hex without the Phase 27 match row.
**How to avoid:** Comparator asserts `read_chip_id_tx_match: true` as a precondition; ban byte-mismatch tasks.
**Warning signs:** Diffs against `command.rs` ReadChipId encoding.

### Pitfall 2: Assuming Config `chip_count=1` Means Full Enumerate Parity
**What goes wrong:** Skip RX-loop work because address_interval already equals 256.
**Why it happens:** Single-chip boards hide count-source bugs numerically.
**How to avoid:** Still implement drain-until-idle + soft-retry + counted handoff; measure `chip_count_source_class` and `enumerate_to_mining_ready_gap_class`.
**Warning signs:** Disposition claims “already matched” without RX-loop markers.

### Pitfall 3: Reopening Falsified Levers
**What goes wrong:** Diagnostic recommends poll/long-block/post-baud/ticket-mask-as-sole-blocker.
**Why it happens:** Older recommenders still list those enums.
**How to avoid:** Hard-fail unit tests if banned labels appear (mirror 28.1.1.4 compare-test).
**Warning signs:** `recommended_investigation` contains any falsified knob.

### Pitfall 4: Soft-Retry vs Fail-Closed Confusion
**What goes wrong:** Rust keeps failing closed on first bad preamble and never drains idle.
**Why it happens:** Current `validate_single_chip_detect_response` is single-shot.
**How to avoid:** Match upstream: `continue` on preamble/CRC; `break` on wrong length / UART error; exit loop on received==0; then compare counted vs expected.
**Warning signs:** Chip detect still “works” but gap_class stays `immediate` while upstream is `drain_idle_like`.

### Pitfall 5: Dropping ASIC-256 Ticket-Mask Wire Parity
**What goes wrong:** A/B accidentally reverts to pool-1000 mask.
**Why it happens:** Mining-ready config churn during handoff patch.
**How to avoid:** Keep `catalog.asic().difficulty()` / ASIC-256; assert `wire_parity_ticket_mask_retained: true` in evidence (D-15).
**Warning signs:** `difficulty_mask_class_rust` leaves `diff_256`.

### Pitfall 6: Early Exit Before 360 s / Missing Detector Gate
**What goes wrong:** False negative on correlate/submit.
**Why it happens:** Default monitor capture is 25 s.
**How to avoid:** `capture-timeout-seconds>=360`, wall clock ≥420 s, `just detect-ultra205` first.
**Warning signs:** Evidence `capture_timeout_seconds` < 360.

### Pitfall 7: Implementing Version-Rolling as Second Patch
**What goes wrong:** Scope creep after RX-loop A/B unchanged.
**Why it happens:** NEXT-AGENT-PROMPT lists version-rolling as next suspect.
**How to avoid:** Disposition only sets `next_hypothesis: version_rolling_negotiation`; no code in this phase (D-07).
**Warning signs:** Stratum `mining.configure` changes in the same PR as enumerate RX.
