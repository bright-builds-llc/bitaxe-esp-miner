## Threat Model

**Assets:** committed redacted evidence, exact parity claims, prior valid Phase 28 roots, local credential/raw-evidence boundaries.

**Trust boundaries:** CLI arguments into typed profiles; source evidence files into the consolidator; staging filesystem into destination promotion; generated slot metadata into validation.

| Threat | Severity | Mitigation | Verification |
| --- | --- | --- | --- |
| Profile spoofing or path-based misclassification | High | Require `OperatorEvidenceProfile` as a parsed `ValueEnum`; remove path/content profile inference. | Tests use misleading directory names and prove the supplied profile controls policy. |
| Generated placeholder masquerades as observation | High | Model `Observed`, `CrossLinked`, `Blocked`, and `Deferred` as distinct dispositions with profile-specific legal uses. | Tests reject generated/cross-linked provenance where observed proof is mandatory. |
| Source/destination traversal, nesting, or symlink escape | High | Canonicalize existing ancestors, reject equal/nested roots and symlink-managed roots, create staging beside destination. | Tests cover equal, child, parent, `..`, and symlink cases. |
| TOCTOU or partial atomic replacement | High | Validate staging completely before one same-filesystem rename sequence; retain previous root until replacement succeeds. | Failure-injection tests hash the previous destination before and after failure. |
| Secret-bearing raw artifact copy | High | Parse allowlisted fields only and render new files containing relative cross-links/category labels. | Sentinel fixtures prove raw files and forbidden tokens never appear in generated output. |
| Overbroad claim promotion | High | Share outcomes are a closed enum; accepted/rejected require exact ASIC-correlation and safe-stop support. | Negative fixtures reject accepted/rejected without required support. |
</threat_model>
