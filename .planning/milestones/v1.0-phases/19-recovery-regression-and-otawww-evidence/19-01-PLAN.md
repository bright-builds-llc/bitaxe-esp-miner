## Trust Boundaries

| Boundary | Description |
| --- | --- |
| CLI input to Phase 19 wrapper | Untrusted device URL, flash evidence JSON, manifest path, image paths, port, output directory, and allow flags enter the helper. |
| Phase 19 wrapper to Phase 16 helper | Phase 19 delegates potentially destructive recovery flows to an existing helper that owns detector, board-info, restore, and safe-state gates. |
| Local raw evidence to committed docs | Raw target and device logs may contain private network or secret values and must be converted to redacted committed artifacts. |
| HTTP upload probe to firmware | OTA and OTAWWW requests can mutate device state or reveal public route behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-19-01 | Elevation of Privilege / Tampering | `scripts/phase19-recovery-otawww-evidence.sh` target handling | mitigate | Accept only origin-only direct targets or trusted board `205` flash-monitor evidence; write `network_scan: disabled`; do not bypass firmware access gates. |
| T-19-02 | Tampering / Repudiation | package and flash identity inputs | mitigate | Require manifest path input, preserve source/reference fields, and leave current package validation to Plan 02 before evidence citation. |
| T-19-03 | Denial of Service | failed-update, large-erase, and interrupted-upload flags | mitigate | Default to pending no-allow evidence; delegate live actions to Phase 16 helper, which reruns `just detect-ultra205` and `espflash board-info`. |
| T-19-04 | Denial of Service / Tampering | whole-www OTAWWW claim | mitigate | Create only gap evidence in Wave 0; do not mark OTAWWW verified from `www.bin`, route presence, static serving, or `Wrong API input`. |
| T-19-05 | Information Disclosure | evidence artifacts | mitigate | Start `redaction_status: pending`; artifact matrix must list present or `absent - not cited` for target, HTTP, serial, detector, board-info, and recovery logs. |
| T-19-06 | Repudiation / Denial of Service | raw destructive commands | mitigate | Wrapper must not contain direct erase, rollback, raw write, or interrupted upload commands; exact allow flags and helper delegation are grep-verifiable. |
