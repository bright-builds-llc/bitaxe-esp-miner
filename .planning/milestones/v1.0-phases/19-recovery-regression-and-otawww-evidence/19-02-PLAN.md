## Trust Boundaries

| Boundary | Description |
| --- | --- |
| Package manifest to evidence docs | Generated package metadata becomes committed release evidence. |
| USB detector to hardware action | Read-only detector output controls whether flash-monitor can run. |
| Flash-monitor output to target lock | Local raw evidence may include private target data that must not be committed. |
| Wi-Fi credential path to flash wrapper | The ignored credential file may be passed by path but must never be inspected or committed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-19-01 | Elevation of Privilege / Tampering | flash-monitor and target provenance | mitigate | Run `just detect-ultra205` before hardware work; proceed only with exactly one port and successful board-info for board `205`. |
| T-19-02 | Tampering / Repudiation | package identity | mitigate | Run `just package`, copy the manifest, run release-gate on the copied manifest, and verify `source_commit` equals current `HEAD` before citation. |
| T-19-03 | Denial of Service | flash-monitor execution | mitigate | Use repo-owned `just flash-monitor`; stop and record blocked evidence if detector, board-info, or wrapper trust fails. No recovery or destructive action runs in this plan. |
| T-19-04 | Denial of Service / Tampering | static administration UI | accept | This plan only captures package/static identity and does not write the whole `www` partition; OTAWWW remains non-claim text. |
| T-19-05 | Information Disclosure | target lock and serial artifacts | mitigate | Commit only redacted target provenance, use `redact-evidence=true`, keep raw developer evidence under `target/`, and record `network_scan: disabled`. |
| T-19-06 | Repudiation / Denial of Service | raw hardware commands | mitigate | Use only repo-owned `just detect-ultra205` and `just flash-monitor`; no raw erase, rollback, write, or interrupted-upload command appears in this plan. |
