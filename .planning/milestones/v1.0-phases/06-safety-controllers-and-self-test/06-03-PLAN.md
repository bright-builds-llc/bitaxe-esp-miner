## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Raw INA260/I2C data -> safety domain | Sensor reads are untrusted and may be stale, missing, cached, non-finite, or unsafe. |
| User/config voltage request -> DS4432U effect plan | Requested millivolts can be invalid or unsafe for Ultra 205 BM1366. |
| Evidence state -> voltage actuation | Incorrect evidence treatment could allow hardware effects without proof. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-06-03-02 | Spoofing | `PowerObservation::from_ina260_sample` | mitigate | Treat missing, read-failed, stale, and non-finite samples as unavailable/faulted instead of fresh telemetry. |

| T-06-03-04 | Repudiation | Hardware verification state | mitigate | Require `SafetyCriticalEvidence::HardwareSmoke` or `HardwareRegression` plus `ArmedWithHardwareEvidence` before any write plan appears. |
