## Trust Boundaries

| Boundary | Description |
| --- | --- |
| operator target input to helper | Explicit `DEVICE_URL` controls live HTTP requests. |
| package/flash identity to live route claims | The copied package manifest and flash-monitor JSON decide whether HTTP probes can support just-flashed evidence. |
| live HTTP responses to committed artifacts | Headers, bodies, and curl errors may contain private device or network values. |
| route evidence to parity claims | Static, recovery, API, WebSocket route-coexistence, OTA route-presence, and OTAWWW gap evidence have different claim scopes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-17-03-target | Spoofing/Tampering | HTTP helper execution | mitigate | Probe only explicit origin-only `DEVICE_URL`; when absent or invalid, write blocked evidence and keep `network_scan: disabled` with no fallback discovery. |
| T-17-03-identity | Tampering/Repudiation | package and flash evidence preflight | mitigate | Require Plan 17-02 package manifest and wrapper flash JSON before live route evidence can support just-flashed claims. |
| T-17-03-redaction | Information Disclosure | HTTP artifacts and redaction review | mitigate | Store selected headers and sanitized body snippets only; redact raw target URLs, private endpoints, IPs, MACs, Wi-Fi/pool/API/NVS secrets, and mark absent artifacts `absent - not cited`. |
| T-17-03-overclaim | Repudiation/Elevation of Privilege | `http-static-api.md` and route artifacts | mitigate | Record `/api/ws` no-upgrade as route coexistence only, empty `/api/system/OTA` as route-presence only, and `/api/system/OTAWWW` as fail-closed gap response. |
</threat_model>
