<threat_model>
## Trust Boundaries

| Boundary | Description |
| --- | --- |
| Firmware logs -> retained buffer -> HTTP/WebSocket | Runtime logs may include operational or secret-adjacent data. |
| Runtime snapshot -> live telemetry WebSocket | Device state is pushed repeatedly to connected clients. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-05-11 | Information Disclosure | log download and `/api/ws` | mitigate | Plan 05 must gate routes; this plan tests no secret logging by new API code and raw chunks only. |
| T-05-12 | Denial of Service | log buffer and WebSocket streams | mitigate | Bound retention, chunk sizes, no-client hibernation, and no-send unchanged telemetry behavior. |
| T-05-13 | Tampering | telemetry diff | mitigate | Structured diff tests prevent malformed or partial update envelopes that AxeOS would merge incorrectly. |
| T-05-14 | Spoofing | WebSocket clients | mitigate | Client admission and private-network/AP gate are enforced in Plan 05; this plan records the contract. |

High-risk block: stop if logs are replayed to new `/api/ws` clients, if `/api/ws/live` omits the full connect update, or if telemetry loops send unchanged frames every cadence tick.
</threat_model>
